September 9th

REPL Driven Development

The Console Plugin's summary on grails.org states it is intended for "runtime application management and debugging". However, it can also be employed during development to interactively experiment with existing application artefacts as well as write new code. Just as Grails' auto-reloading of artefacts boosts developer productivity by shortening the feedback cycle, interacting with your code in realtime shortens the cycle and increases productivity even further.
In this example-heavy, mostly live-coding presentation we'll examine how using the Console Plugin for development leads us toward small, functional, composable methods and an expressive, easy to reason about, maintainable and testable code base.

Speaker: David Kuster
